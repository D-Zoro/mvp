┌─ CRIT-01: ORDER QUANTITY RACE CONDITION ─────────────────────────────────┐
│                                                                             │
│ PROBLEM:                                                                    │
│   Thread A reads Book.quantity = 2 ✓                                      │
│   Thread B reads Book.quantity = 2 ✓                                      │
│   Both pass stock check                                                    │
│   Thread A: quantity = 2 - 2 = 0 ✓                                        │
│   Thread B: quantity = 0 - 2 = -2 ✗ OVERSELL!                            │
│                                                                             │
│ SOLUTION: SELECT ... FOR UPDATE (lock row during transaction)            │
│                                                                             │
│ FILES:   backend/app/repositories/order.py                               │
│          backend/app/services/order_service.py                           │
│          backend/app/main.py (error handling)                            │
│                                                                             │
│ EFFORT:  4-6 hours (dev: 2-3h, test: 1-2h, review: 1h)                  │
│                                                                             │
│ TEST:    Load test with 5+ concurrent orders on same book               │
│          Verify: qty never negative, no oversells                        │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

┌─ CRIT-02: STRIPE WEBHOOK DEDUPLICATION ──────────────────────────────────┐
│                                                                             │
│ PROBLEM:                                                                    │
│   Webhook delivered → order marked PAID ✓                                 │
│   Network error (no ACK to Stripe)                                        │
│   Stripe retries webhook with same event → marked PAID again             │
│   Result: Audit confusion, potential double-charge logic bugs            │
│                                                                             │
│ SOLUTION: Cache processed event IDs in Redis (24h TTL)                  │
│                                                                             │
│ FILES:   backend/app/services/payment_service.py                         │
│          backend/app/core/rate_limiter.py (Redis helpers)               │
│          backend/tests/integration/test_payments_api.py                  │
│                                                                             │
│ EFFORT:  2.5-3.5 hours (dev: 1-2h, test: 1h, review: 0.5h)              │
│                                                                             │
│ TEST:    Send same webhook event twice                                    │
│          Verify: First processed, second returns cached result           │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

┌─ CRIT-03: JWT SECRET ROTATION ───────────────────────────────────────────┐
│                                                                             │
│ PROBLEM:                                                                    │
│   Default SECRET_KEY is public in docs                                   │
│   No rotation mechanism (change SECRET_KEY → all tokens invalid)         │
│   On breach, must invalidate all sessions (high churn)                   │
│                                                                             │
│ SOLUTION: JWT key versioning (kid claim + 30-day deprecation window)     │
│   - Add kid header to tokens                                             │
│   - Accept current + deprecated keys (within 30d TTL)                   │
│   - Rotate: generate new secret, gradually migrate tokens               │
│                                                                             │
│ FILES:   backend/app/core/keys.py (NEW)                                  │
│          backend/app/core/security.py                                    │
│          backend/app/core/config.py                                      │
│          backend/app/core/dependencies.py                                │
│                                                                             │
│ EFFORT:  4-6 hours (dev: 2-3h, test: 1-2h, review: 1h)                  │
│                                                                             │
│ TEST:    Create token with v1 key                                         │
│          Rotate to v2, verify v1 still works (within 30d)               │
│          After 30d, verify v1 rejected                                   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
